Overview

From ProcessMaker version 2.5.1 the possibility to apply Hotfixes to solve certain bugs was added. These Hotfixes will be made available only to ProcessMaker Enterprise Customers.

In addition, it is worth mentioning that these Hotfixes will be cumulative, in other words: if the first Hotfix comes with 5 bugfixes, the second Hotfix will include those 5 bugfixes plus some new ones. Therefore it is not necessary to apply all Hotfixes, since the latest Hotfix will have all previously available bugtfixes, unless you need the resolution of a very specific bug.

Note: These bugfixes will be only applied to the ProcessMaker Source Code and not to the ProcessMaker Backend Database, so there should be no danger of data loss or modification after or during the Hotfix installation procedure.
The Hotfixes can only be applied from the command line terminal on the ProcessMaker server.

In Linux/UNIX ProcessMaker servers:
Open a terminal and change to the directory where ProcessMaker is installed before executing the hotfix. For example:

cd /opt/processmaker
./processmaker hotfix-install

In Windows ProcessMaker servers:
Open the "Command Prompt", by going to Start > Run and entering "cmd". Then, change to the directory where ProcessMaker is installed before executing the hotfix. For example:

cd C:\Users\USERNAME\AppData\Roaming\ProcessMaker-2_X_X\processmaker
./processmaker hotfix-install

For more detailed information about applying the Hotfix please check the complete documentation
