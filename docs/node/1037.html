Entrust

This section discusses how to set up the SAML Auth plugin with Entrust and how to log on to ProcessMaker.

After Installing and enabling the SAML Auth plugin, follow these steps:

  1. Click here to create a new account on Entrust. If you already have an account, jump to step 6.

  2. Click START FREE TRIAL.

  3. A new new tab opens in the browser. Fill in the new account information.

  4. After to fill in the new account information, click the START TRIAL button.

  5. An email is sent to the set account. Click on the link provided to access the new user.

  6. Log on to your Entrust account.

  7. The application requires the One Time Password. If it does not send it automatically, click on the Resend OTP using SMS.

  8. A message and an email with the OTP code are sent to your account.

  9. After logged on, click on Users to verify the status of the registered users.

  10. Click a row in the list to see the details or edit the user configuration.

  11. Check the user details and make sure the State is active.

  12. Go back to the main menu, and then click Applications.

  13. Click the icon to add a new application.

  14. Search and select the Generic SAML Application.

  15. After filling in the Application Name and the Application Description, click NEXT button.

  16. Copy the following values from ProcessMaker Service Provider section (first image below) to the Configure SAML section in Entrust:

    Note: Field hints provided by Entrust also provide a guide about how to match values.

    • Assertion Consumer Service to Assertion Consumer Service URL.
    • Entity ID to Service Provider Entity ID (Issuer).
    • Single Logout Service to Single Logout Service URL.

  17. Select the SAML NameID Attribute and the SAML NameID Encoding Format. Leave the other attributes by default.

  18. In ProcessMaker, select the Configurations tab, fill the Idp Attribute Name field with the attribute mapped in Entrust.

  19. Click DONE to complete the application configuration.

  20. In the Applications List page, click on SAML CONFIGURATION.

  21. A dialog opens with the SAML configuration required by ProcessMaker in the Identity Provider section. Copy the following fields from Entrust (first image below) and then click Update Configuration in ProcessMaker:

    • Entity ID to Entity ID.
    • Single Sign-On URL to Single Sign-On Service.

  22. Go back to the Entrust Applications List page, and then click on SIGNING CERTIFICATES.

  23. Click the icon.

  24. Select the Certificate option, and then click EXPORT button.

  25. In ProcessMaker Configurations tab, click Upload New Certificate and then click Browse.

  26. Select the downloaded Certificate file, and then click Save.

  27. Click on Update Configuration.

To be redirected to Entrust login, in your ProcessMaker domain, enter the Workspace name and then click Login.

This redirects you to your Entrust domain, then enter your Entrust credentials. You are now logged on to ProcessMaker via Entrust.
